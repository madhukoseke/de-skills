## Summary
The DAG has reliability issues that should be corrected before production.

## Decision
Replace `datetime.now()` scheduling usage, avoid bare `insert` patterns that can duplicate data, and configure retry with exponential backoff.

## Rationale
Dynamic timestamps can cause unstable scheduling behavior, and non-idempotent inserts create duplicate records on rerun.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Strict reliability fixes | Better stability | More implementation effort | Production pipelines |
| Keep as-is | Fast delivery | Higher incident risk | Temporary prototypes |

## Next Steps
1. Move to scheduler macros instead of datetime.now.
2. Make writes idempotent.
3. Add retry and backoff configuration.
