## Summary
Define a core data quality control set for the orders fact pipeline.

## Decision
Implement freshness, completeness, and uniqueness checks with explicit failure handling.

## Rationale
These controls catch the most common and damaging data issues before downstream consumption.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Fail-fast checks | Higher trust in data | More pipeline interruptions | Critical reporting |
| Alert-only checks | Fewer failed runs | Potential bad data leakage | Early adoption phase |
