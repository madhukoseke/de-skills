## Summary
The table should be created with date-based partitioning and optional clustering for common filters.

## Decision
Use `CREATE TABLE` with a partition column and clustering/indexing on high-selectivity query columns. Include a cost estimate from expected scan size.

## Rationale
Partitioning reduces scanned data, and clustering or indexing improves selective query performance.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Partition + cluster | Balanced performance | More design work | High-volume analytics |
| Single wide table | Fast to start | Cost/performance issues | Prototype only |
