## Summary
A daily batch architecture is enough for this CRM ingestion use case.

## Decision
Use a scheduled extraction job from Salesforce into staging and then load curated tables for analytics.

## Rationale
The SLA does not require real-time processing, so a simple batch setup is easier to maintain.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Daily batch | Simple and stable | Less fresh data | Standard reporting workloads |
| Streaming | Faster freshness | Higher complexity | Low-latency dashboards |
