## Summary
Use incremental processing with a watermark for late records.

## Decision
Apply merge-based updates over a recent correction window and dedup repeated events.

## Rationale
This supports late-arriving data while keeping the load idempotent.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Merge incremental | Corrective updates | More complexity | Event pipelines |
| Append only | Simpler operations | Poor late handling | Immutable logs |
