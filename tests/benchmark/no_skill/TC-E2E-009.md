## Summary
An incremental dbt model with tests is appropriate for this orders use case.

## Decision
Use dbt incremental materialization, enforce uniqueness, and orchestrate with airflow for scheduling and retries.

## Rationale
Incremental modeling improves runtime and cost while retaining maintainable SQL transformations.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Incremental model | Efficient for large data | Needs robust merge logic | Growing fact tables |
| Full refresh | Simpler logic | Expensive at scale | Small datasets |
