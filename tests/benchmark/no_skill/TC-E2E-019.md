## Summary
Streaming correctness needs durable checkpointing and dedup logic.

## Decision
Use checkpoint storage, event dedup, and exactly-once processing where possible.

## Rationale
These controls improve recovery and reduce duplicate output.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Exactly-once path | Higher correctness | More overhead | Critical streams |
| At-least-once path | Simpler operations | Needs dedup later | Non-critical streams |
