## Summary
Historical loads should run as controlled backfill windows with idempotent logic.

## Decision
Process data in windows, throttle batch pace, and validate reconciliation checks.

## Rationale
Windowed backfill improves retry safety and operational control.

## Trade-offs
| Option | Pros | Cons | When to Use |
|--------|------|------|-------------|
| Windowed backfill | Better recovery | More orchestration | Large histories |
| Single pass | Simpler run plan | Harder to recover | Small data volumes |
